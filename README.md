В ходе выполнения задания я научился изолировать разработку новых функций в отдельных ветках (feature/*), что позволяет вести параллельную работу над задачами. Локальное слияние (merge) удобно использовать для быстрой интеграции небольших изменений, тогда как Pull Request обеспечивает прозрачность и контроль при объединении более крупных доработок. Экспериментальные ветки позволяют безопасно проверять гипотезы, не засоряя основную кодовую базу, а их своевременное удаление поддерживает порядок в проекте.
